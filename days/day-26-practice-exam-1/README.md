## Exam Tips

### General Strategy

1. **Read the entire question** - Don't skim
2. **Identify keywords** - "most appropriate", "best practice", "most efficient"
3. **Eliminate wrong answers** - Narrow down to 2 options
4. **Consider edge cases** - Think about limitations and constraints
5. **Trust your preparation** - You've completed 25 days of intensive study

### Common Traps

- **"All of the above"** - Rarely correct, verify each option
- **Absolute words** - "always", "never", "must" are often wrong
- **Similar options** - Look for subtle differences
- **Overthinking** - Don't read too much into questions

### Time Management

- **Don't get stuck** - If unsure after 2 minutes, flag and move on
- **Keep pace** - Check time every 15 questions
- **Save time for review** - Aim to finish with 10 minutes remaining

## Key Concepts to Remember

Quick reference for common exam topics:

### Data Movement
- **Snowpipe**: Auto-ingest from cloud storage, uses serverless compute
- **Streams**: CDC with METADATA$ACTION, METADATA$ISUPDATE, METADATA$ROW_ID
- **Tasks**: CRON scheduling, DAGs, SYSTEM$STREAM_HAS_DATA()
- **External Tables**: Read-only, query data in cloud storage

### Performance
- **Clustering**: Best for high-cardinality columns, check clustering_depth
- **Materialized Views**: Auto-maintained, cost vs. performance trade-off
- **Warehouse Sizing**: T-shirt sizes (XS to 6XL), multi-cluster for concurrency
- **Result Cache**: 24-hour TTL, exact query match required

### Security
- **RBAC**: Role hierarchy, future grants, OWNERSHIP vs. USAGE
- **Masking**: Column-level, role-based visibility
- **Row Access**: Row-level filtering, policy per table
- **Time Travel**: 0-90 days (Enterprise), Fail-safe 7 days (non-recoverable)

### Monitoring
- **ACCOUNT_USAGE**: 45-min to 3-hour latency, 365-day retention
- **INFORMATION_SCHEMA**: Real-time, limited retention
- **Query Profile**: Identify bottlenecks, partition pruning, spilling

### Advanced
- **Stored Procedures**: JavaScript/SQL/Python, EXECUTE AS CALLER/OWNER
- **UDFs**: Scalar vs. Table (UDTF), MEMOIZABLE for caching
- **Snowpark**: DataFrame API, lazy evaluation, pushdown optimization
